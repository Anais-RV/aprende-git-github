# Autocheck — Commits bien escritos

## Preguntas

1. **¿Qué pregunta debes hacerte antes de escribir un mensaje de commit?**
   Escríbela con tus palabras.

2. **¿Qué tipo de commit usarías en cada caso?**
   - Añades una nueva sección a la documentación
   - Corriges un error tipográfico en el código
   - Actualizas el archivo `.gitignore`
   - Añades una nueva función al proyecto

3. **¿Cuál es la diferencia entre el título y el cuerpo de un commit?**
   ¿Cuándo tiene sentido escribir el cuerpo?

4. **¿Cuándo está bien usar `git commit --amend`? ¿Cuándo no?**

5. **Mira tu historial con `git log --oneline`.**
   ¿Todos tus commits son comprensibles sin necesidad de abrir los archivos?

---

## Señales de que lo tienes controlado

- Tus commits siguen la convención `tipo: descripción`
- No tienes commits con mensajes como "cambios", "wip" o "arreglado"
- Puedes leer tu historial y entender la evolución del proyecto
- Sabes cuándo usar `--amend` y cuándo no
