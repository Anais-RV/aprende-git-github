# Referencia de Markdown

Este tema no tiene comandos de terminal. En su lugar, aquí tienes una referencia rápida de Markdown para usar en tus READMEs y archivos de documentación.

---

## Títulos

```markdown
# H1 — título principal (uno por archivo)
## H2 — sección
### H3 — subsección
#### H4 — subapartado
```

---

## Texto

```markdown
**negrita**
*cursiva*
~~tachado~~
`código en línea`
```

---

## Listas

```markdown
- Elemento
- Otro elemento
  - Subelemento (dos espacios de indentación)

1. Primer paso
2. Segundo paso
3. Tercer paso
```

---

## Código

````markdown
```bash
git commit -m "feat: nueva funcionalidad"
```

```python
print("Hola mundo")
```

```javascript
console.log("Hola mundo")
```
````

---

## Enlaces e imágenes

```markdown
[Texto del enlace](https://url.com)

![Texto alternativo de la imagen](ruta/imagen.png)

<!-- Imagen con enlace -->
[![Alt](imagen.png)](https://url.com)
```

## Tablas

```markdown
| Columna A | Columna B | Columna C |
|-----------|-----------|-----------|
| Dato 1    | Dato 2    | Dato 3    |
| Dato 4    | Dato 5    | Dato 6    |
```

---

## Citas y notas

```markdown
> Esto es una cita o nota destacada.

> [!NOTE]
> GitHub renderiza este bloque como una nota visual.

> [!WARNING]
> Y este como una advertencia.
```

---

## Separadores y espaciado

```markdown
---
```

Una línea con tres guiones crea una línea horizontal de separación.
