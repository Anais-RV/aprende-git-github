# Ejercicio — README y documentación

## Parte 1: Evalúa el README actual de tu repositorio

Abre `practica-git` en GitHub y lee su README (o la ausencia de él).

Responde honestamente:
- ¿Alguien que no conoce tu proyecto podría entender para qué sirve?
- ¿Sabe ese alguien cómo empezar a usarlo?
- ¿Hay información desactualizada o que falta?

---

## Parte 2: Reescribe el README de tu repositorio de práctica

Crea o reescribe el `README.md` de tu repositorio `practica-git` con esta estructura mínima:

```markdown
# practica-git

[Una frase que describa para qué usaste este repositorio]

## ¿Qué hay aquí?

[Breve descripción de los archivos y carpetas del repositorio]

## Cómo usé este repositorio

[Explica qué practicaste y en qué orden]

## Lo que aprendí

[Un párrafo sobre los conceptos más importantes que trabajaste]
```

No copias el texto de arriba tal cual — escríbelo con tus palabras y adaptado a lo que realmente hiciste.

---

## Parte 3: Aplica al menos 5 elementos de Markdown distintos

Asegúrate de que tu README use:
- Al menos 2 niveles de títulos
- Texto en negrita o cursiva
- Una lista (puede ser ordenada o no)
- Un bloque de código con el lenguaje especificado
- Un enlace a un recurso externo

---

## Parte 4: Haz commit del README mejorado

```bash
git add README.md
git commit -m "docs: reescribir README con estructura clara"
git push
```

---

## Parte 5 (opcional): Crea un CONTRIBUTING.md

Si tienes tiempo, crea un archivo `CONTRIBUTING.md` básico que explique cómo alguien podría contribuir a tu repositorio de práctica. Puedes inspirarte en el ejercicio del tema 08 (fork y PR).

---

## Resultado esperado

- Tu `practica-git` tiene un README que alguien ajeno podría entender
- Usas correctamente al menos 5 elementos de Markdown
- El README está actualizado y reflejas lo que realmente contiene el repositorio
