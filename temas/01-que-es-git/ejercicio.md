# Ejercicio — ¿Qué es Git?

Este primer ejercicio no requiere escribir código. Es un ejercicio de **reflexión y verificación** antes de ponernos manos a la obra.

---

## Paso 1: Comprueba que tienes Git instalado

Abre una terminal (en Windows puedes usar Git Bash, PowerShell o el terminal integrado de VS Code) y ejecuta:

```bash
git --version
```

- Si ves un número de versión → estás listo. Apunta la versión que tienes.
- Si ves un error → visita [git-scm.com](https://git-scm.com) e instala Git.

---

## Paso 2: Reflexión personal

Antes de continuar, responde mentalmente (o por escrito si lo prefieres) estas preguntas:

1. ¿Tienes algún proyecto o carpeta donde hayas creado archivos del tipo `_v2`, `_final`, `_definitivo`?
2. ¿Alguna vez perdiste trabajo porque sobreescribiste un archivo sin querer?
3. ¿Qué te gustaría poder hacer con tus proyectos que ahora no puedes?

No hay respuestas correctas o incorrectas. El objetivo es que conectes el problema con la solución antes de aprender los comandos.

---

## Paso 3: Explora la ayuda de Git

Ejecuta:

```bash
git help
```

Observa los comandos listados. No hace falta que los entiendas todos ahora. Simplemente familiarízate con que Git tiene su propia documentación accesible desde la terminal.

---

## Resultado esperado

Al terminar este ejercicio deberías poder:
- Confirmar que Git está instalado en tu máquina
- Explicar con tus propias palabras qué problema resuelve Git
