# Autocheck — ¿Qué es Git?

Responde estas preguntas para comprobar que tienes claro el tema. Si alguna te cuesta, vuelve a la explicación.

---

## Preguntas

1. **¿Qué es un sistema de control de versiones?**
   Intenta explicarlo con tus propias palabras, sin usar la palabra "versiones".

2. **¿Cuál es la diferencia entre Git y GitHub?**
   ¿Podría usar Git sin internet? ¿Y GitHub sin Git?

3. **¿Por qué es problemático guardar versiones con nombres como `archivo_final_v2`?**
   Piensa en al menos dos problemas concretos que genera esa práctica.

4. **¿Dónde vive Git: en tu máquina o en internet?**

5. **¿Para qué tipo de proyectos tiene sentido usar Git?**
   ¿Solo para código? ¿O también para otros tipos de archivos?

---

## Señales de que lo tienes controlado

- Puedes explicar Git a alguien que nunca lo ha usado
- Entiendes que Git y GitHub son cosas distintas
- Tienes Git instalado y funciona correctamente en tu máquina
